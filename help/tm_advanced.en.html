TimeMap - Advanced

The advanced algorithm maps as closely as possible with the TimeMap endpoint from the Travel Time Platform API. Thus, it allows maximum flexibility, but requires a good understanding of the API to use correctly.

It is a good idea to get familiar with the simplified algorithm first, as it already covers most of the simple use cases.

Once you feel confident enough to dig deeper, you will find an extensive explanation of all inputs on the API documentation page.

There are two main differences with the API :

  • The API limits queries to 10 searches at a time. The plugin therefore runs multiple queries if the input dataset is bigger than 10 points.
  • You cannot provide a list of searches for which to compute union/intersection ; instead, they are computed on ALL results
